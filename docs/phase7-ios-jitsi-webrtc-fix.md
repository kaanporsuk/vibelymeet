# Phase 7 — iOS JitsiWebRTC Build Blocker Fix

## 1. Exact root cause

**Incomplete WebRTC.xcframework on disk:** The pod `JitsiWebRTC` (dependency of `@daily-co/react-native-webrtc`) provides `WebRTC.xcframework`. Its `Info.plist` declares three slices:

- `ios-arm64` (device)
- `ios-arm64_x86_64-simulator` (simulator)
- `macos-arm64_x86_64`

Only **`ios-arm64_x86_64-simulator`** was present under `ios/Pods/JitsiWebRTC/WebRTC.xcframework/`. The **`ios-arm64`** (device) slice was **missing**. The [CP] Copy XCFrameworks build phase selects the slice by destination (device vs simulator); for a device build it looks for `WebRTC.xcframework/ios-arm64/*`, which did not exist.

**Cause:** Stale or partial CocoaPods install / cache (or a CocoaPods CDN artifact that sometimes omits the device slice). Not a signing or podspec bug.

---

## 2. Exact commands run

```bash
# Inspection
ls -la apps/mobile/ios/Pods/JitsiWebRTC/WebRTC.xcframework
cat apps/mobile/ios/Pods/JitsiWebRTC/WebRTC.xcframework/Info.plist
cat apps/mobile/ios/Pods/Target\ Support\ Files/JitsiWebRTC/JitsiWebRTC-xcframeworks.sh

# Fix: Podfile updated (see below), then clean pod install
cd apps/mobile/ios
LANG=en_US.UTF-8 pod install

# Verify slice present
ls -la apps/mobile/ios/Pods/JitsiWebRTC/WebRTC.xcframework/

# Verify build gets past JitsiWebRTC
cd apps/mobile && npx expo run:ios --no-bundler --configuration Debug
```

---

## 3. Exact files changed

| File | Change |
|------|--------|
| **`apps/mobile/ios/Podfile`** | (1) Added `require 'fileutils'`. (2) In the existing `post_install` block, added a check: if `JitsiWebRTC/WebRTC.xcframework` exists but `WebRTC.xcframework/ios-arm64` does not, download `https://github.com/jitsi/webrtc/releases/download/v124.0.2/WebRTC.xcframework.zip`, extract only the `ios-arm64` slice, and copy it into `Pods/JitsiWebRTC/WebRTC.xcframework/ios-arm64`. This restores the device slice when the pod install leaves it missing. |

No other files were changed. `Pods/`, `Podfile.lock`, and `build/` are not in git; they are regenerated by `pod install` and `expo run:ios`.

---

## 4. Whether iOS build now gets past JitsiWebRTC

**Yes.** After the fix:

- A full `pod install` was run. The JitsiWebRTC pod was installed; this time the xcframework under `Pods/JitsiWebRTC/WebRTC.xcframework/` contained all three slices (`ios-arm64`, `ios-arm64_x86_64-simulator`, `macos-arm64_x86_64`). So either the clean install pulled a complete artifact or the post_install restore would have run if the slice were still missing.
- `npx expo run:ios --no-bundler --configuration Debug` was started. The log shows:
  - `› Executing @daily-co/react-native-webrtc Pods/JitsiWebRTC » [CP] Copy XCFrameworks` **completed without error.**
  - The build continued to later targets (e.g. OneSignalXCFramework, expo-image). So the iOS build **gets past** the JitsiWebRTC [CP] Copy XCFrameworks step.

## 5. If still blocked (next blocker after JitsiWebRTC)

If the same “selected slice ios-arm64” error appears again:

1. Ensure the Podfile in `apps/mobile/ios` still contains the `post_install` block that restores the `ios-arm64` slice (see §3).
2. Remove Pods and reinstall:
   ```bash
   cd apps/mobile/ios
   rm -rf Pods Podfile.lock build
   LANG=en_US.UTF-8 pod install
   ```
3. Confirm the slice exists:
   ```bash
   ls apps/mobile/ios/Pods/JitsiWebRTC/WebRTC.xcframework/ios-arm64
   ```
4. If `ios-arm64` is still missing after `pod install`, the post_install will download the zip and copy the slice; check that `curl` and `unzip` are available and that the GitHub URL is reachable.

If a **different** build error appears after JitsiWebRTC (e.g. signing, another pod, or linking), that is a separate blocker and should be handled as a new issue.

---

## Rebuild / prebuild note

The Podfile is under `ios/`, which Expo can regenerate with `npx expo prebuild --platform ios --clean`. If you run prebuild with `--clean`, re-apply the Podfile changes (the `require 'fileutils'` and the JitsiWebRTC post_install block) or add them via an Expo config plugin so they survive prebuild.
